Supabase Production Kurulumu
Supabase
⚠️
Mevcut dev projenizi production için kullanmayın. Yeni bir proje oluşturun — data, RLS ve bakım açısından temiz başlamak kritiktir.
📋 Yeni Proje Oluşturma
🌍
Bölge: eu-central-1 (Frankfurt)
KVKK uyumu için Türkiye'ye en yakın Supabase bölgesi. supabase.com/dashboard → New Project
🔑
Güçlü DB Parolası
En az 32 karakter. Kaydedin — sonra değiştirmek karmaşıklaşır.
💲
Plan: Pro ($25/ay)
Günlük backup, 8GB storage, Realtime için gerekli. Free plan production için yeterli değil.
🗃️ Migration'ları Uygulama
bash
# 1. Supabase CLI kurulumu (homebrew / npm) brew install supabase/tap/supabase # veya: npm install -g supabase # 2. Prod projesine bağlan supabase login supabase link --project-ref PROJE_ID # 3. Migration'ları uygula (001 → 010) supabase db push # 4. Seed verilerini YÜKLEMEYİN — sadece prod schema # supabase db reset ← BU KOMUTU ÇALIŞTIRMAYIN (tüm datayı siler) # 5. Type'ları üret (opsiyonel — local geliştirme için) supabase gen types typescript --project-id PROJE_ID --schema public > types/database.ts
🔐 Supabase Auth Ayarları
📧
Auth → URL Configuration
Site URL: https://ev-yemekleri.com
Redirect URLs: https://ev-yemekleri.com/**
📱
Phone Auth → Netgsm entegrasyonu
Auth → Providers → Phone → Custom SMS Provider seçin. Netgsm webhook URL'ini girin.
⏰
OTP süresi: 10 dakika
Auth → Settings → OTP Expiry → 600 saniye.
🗄️ Storage Bucket Ayarları
sql — Supabase SQL Editor'da çalıştırın
-- Bucket'ları public/private olarak doğrulayın SELECT name, public FROM storage.buckets; -- Beklenen çıktı: -- avatars | false (imzalı URL ile erişim) -- menu-photos | true (CDN üzerinden herkese açık) -- chef-documents | false (admin erişimi) -- blog-images | true (herkese açık) -- Storage CDN (önbellek) — Production'da etkinleştirin -- Dashboard → Storage → Settings → CDN caching → ON
🔒 RLS Doğrulaması
sql
-- Tüm tablolarda RLS aktif olduğunu doğrulayın SELECT tablename, rowsecurity FROM pg_tables WHERE schemaname = 'public' ORDER BY tablename; -- Tüm satırlar "rowsecurity = true" olmalı -- false olan tablo varsa: ALTER TABLE <tablo_adi> ENABLE ROW LEVEL SECURITY;
⏰ pg_cron Aktivasyonu
sql
-- Cron job'ların aktif olduğunu kontrol edin SELECT jobname, schedule, active FROM cron.job; -- Beklenen çıktı: -- reset_daily_stock | 0 3 * * * | true -- archive_old_orders | 0 2 * * 0 | true -- cleanup_expired_otps | */15 * * * * | true -- ... (010_cron.sql'deki tüm job'lar)

Vercel'de proje oluşturun
vercel.com/new → GitHub repo'nuzu seçin → Framework: Next.js (otomatik algılanır)
🌍
Bölge: fra1 (Frankfurt)
Project Settings → Functions → Serverless Function Region → fra1. Supabase ile aynı bölge — gecikme minimuma iner.
⚙️ Vercel CLI ile Deploy
bash
# Vercel CLI npm install -g vercel # İlk bağlantı (browser açılır) vercel login vercel link # Env değişkenlerini import et (.env.local → Vercel) vercel env pull .env.local # Vercel → local (preview env) # Production'a deploy vercel --prod # Deploy loglarını izle vercel logs --follow
🏗️ Build Optimizasyonu
next.config.ts — eklemeler
const nextConfig = { // Mevcut ayarlara ekleyin: // Bundle analyzer (sadece geliştirmede) experimental: { optimizePackageImports: ['@supabase/supabase-js'], }, // Prod'da source map kapatın productionBrowserSourceMaps: false, // Sıkıştırma compress: true, // Edge Runtime için paketleri dışla serverExternalPackages: ['iyzipay'], }

Domain & SSL
DNS
🌐 Vercel'e Domain Bağlama
bash
# Vercel CLI ile domain ekle vercel domains add ev-yemekleri.com vercel domains add www.ev-yemekleri.com # DNS Provider'ınızda (Cloudflare / GoDaddy vb.): # A @ 76.76.19.19 (Vercel IP) # CNAME www cname.vercel-dns.com # SSL otomatik olarak Let's Encrypt ile sağlanır. # DNS propagasyonu 5-30 dakika sürebilir. # Doğrulama: vercel domains inspect ev-yemekleri.com
🔒 Güvenlik Başlıkları (next.config.ts zaten içeriyor)
🛡️
HSTS (Strict-Transport-Security)
Vercel otomatik olarak HTTPS zorlar. Ek HSTS başlığı eklemek isterseniz next.config.ts headers() içine ekleyin.
🌐
www → apex redirect
Vercel Dashboard → Project → Domains → www.ev-yemekleri.com → Redirect to ev-yemekleri.com

İyzico Production Geçişi
⚠️
İyzico Marketplace hesabı açmak 5-10 iş günü sürebilir. Önceden başvurun.
📋 Başvuru Süreci
1️⃣
İyzico Marketplace Başvurusu
merchant.iyzipay.com → Kayıt Ol → Şirket bilgilerini girin (vergi no, IBAN, faaliyet belgesi)
2️⃣
Sub Merchant (Aşçı) Kaydı API'si
Her aşçı onaylandığında otomatik sub merchant oluşturulur. profil-ayarlari sayfasında IBAN girildiğinde tetiklenir. İyzico dönen subMerchantKey → chef_profiles.iyzico_sub_merchant_key'e kaydedilir.
3️⃣
Callback URL İyzico Panelinde de Girilmeli
İyzico Dashboard → Entegrasyon → Callback URL: https://ev-yemekleri.com/api/payments/callback
4️⃣
Komisyon Oranı: %10
Siparişlerde platform_fee = subtotal * 0.10 otomatik hesaplanıyor. İyzico panel ödeme planında da aynı oran girilmeli.
